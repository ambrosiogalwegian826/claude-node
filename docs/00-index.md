# Documentation Index

This documentation set is written to make the project easy to understand from three different angles:

1. **What the project is**
2. **What the current code actually does**
3. **What should happen next without making the project too broad**

## Reading order

- [`01-positioning.md`](01-positioning.md) — project identity, scope, and non-goals
- [`02-architecture.md`](02-architecture.md) — internal architecture and design boundaries
- [`03-api-reference.md`](03-api-reference.md) — reference for the current public API
- [`04-protocol.md`](04-protocol.md) — `stream-json` protocol notes and runtime behavior
- [`05-development.md`](05-development.md) — repository workflow, testing, and contribution guidance
- [`06-roadmap-and-limitations.md`](06-roadmap-and-limitations.md) — current gaps and the recommended next steps

## Important documentation rule

Throughout this doc set, we intentionally separate:

- **current reality**
- **recommended design direction**
- **explicit non-goals**

This matters because the repository is already useful, but still early. Good documentation should reduce ambiguity, not create it.
