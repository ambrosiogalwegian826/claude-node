# API Reference

This reference describes the **current** public API exposed by the repository.

## Public imports

```python
from claude_node import (
    ClaudeController, ClaudeMessage, MultiAgentRouter, AgentNode,
)
from claude_node.exceptions import (
    ClaudeError, ClaudeBinaryNotFound, ClaudeStartupError,
    ClaudeTimeoutError, ClaudeSendConflictError,
)
from claude_node.runtime import check_claude_available, ClaudeRuntimeInfo
```

## `ClaudeMessage`

Represents a parsed JSON message emitted by the Claude CLI stream.

### Core fields

- `type: str`
- `subtype: str`
- `raw: dict`

### Message classification helpers

- `is_init`
- `is_result`
- `is_result_ok`
- `is_result_error`
- `is_api_error`
- `truly_succeeded`
- `is_task_event`
- `is_assistant`
- `is_tool_result`

### Content helpers

- `result_text`
- `session_id`
- `assistant_texts`
- `tool_calls`
- `tool_results`
- `cost_usd`
- `num_turns`

### Notes

- `is_api_error` is intentionally defensive because some practical error conditions may still surface as `type=result, subtype=success` with an error-like text payload.
- `tool_results` are extracted from `user` messages generated by the CLI after tool execution.

## `ClaudeController`

Controls one persistent Claude CLI subprocess.

### Constructor

```python
ClaudeController(
    system_prompt: str = "",
    append_system_prompt: str = "",
    tools: list[str] = None,
    allowed_tools: list[str] = None,
    disallowed_tools: list[str] = None,
    permission_mode: str = None,
    skip_permissions: bool = False,
    bare: bool = False,
    resume: str = None,
    continue_session: bool = False,
    model: str = None,
    cwd: str = None,
    add_dirs: list[str] = None,
    on_message: Callable[[ClaudeMessage], None] = None,
    transcript_path: str = None,
)
```

### Lifecycle

#### `start(wait_init_timeout: float = 0) -> bool`
Starts the subprocess. If `wait_init_timeout > 0`, waits for a `system/init` message.

#### `stop(timeout: float = 5.0)`
Stops the subprocess gracefully, then kills if needed.

### Sending and waiting

#### `send(text: str, timeout: float = 60.0) -> Optional[ClaudeMessage]`
Sends one user message and waits for the next `result` message.

#### `send_nowait(text: str)`
Sends a message without waiting for completion.

#### `wait_for_result(timeout: float = 60.0, start_index: int = 0) -> Optional[ClaudeMessage]`
Waits for a `result` message, usually after `send_nowait()`.

#### `wait_for_tool_use(tool_name: str, timeout: float = 30.0, start_index: int = 0) -> Optional[dict]`
Waits for a specific `tool_use` block.

### Introspection helpers

#### `get_messages() -> list[ClaudeMessage]`
Returns parsed messages collected so far.

#### `get_init_message() -> Optional[ClaudeMessage]`
Returns the first `system/init` message if present.

#### `get_available_tools() -> list[str]`
Extracts the tool list from the init message.

#### `get_available_skills() -> list[str]`
Extracts slash commands from the init message.

#### `get_stderr() -> list[str]`
Returns captured stderr lines.

#### `get_transcript_path() -> str | None`
Returns the configured transcript file path, or `None` if not set.

#### `get_tool_errors(start_index: int = 0) -> list[dict]`
Returns all failed `tool_result` blocks after a given index.

#### `send_checked(text: str, timeout: float = 60.0) -> Tuple[Optional[ClaudeMessage], list[dict]]`
Convenience helper returning `(result, tool_errors)`.

### Session forking

#### `fork() -> ClaudeController`
Creates a new controller resuming the current session. The new controller is returned stopped; the caller must call `start()` on it.

Raises `RuntimeError` if no session is established (controller not started or start failed).

### Properties

- `pid`
- `alive`
- `session_id`

### Important current behavior

- `send()` timeout returns `None` (partial exception integration — `ClaudeTimeoutError` not yet raised).
- `send()` and `send_nowait()` use `_send_lock` to serialize concurrent calls; conflicts raise `ClaudeSendConflictError`.
- `send_nowait()` + `wait_for_result()` correctly releases the lock on all exit paths.

### Transcript / JSONL export

When `transcript_path` is set in the constructor, every JSON line read from stdout is appended to the specified JSONL file. This provides a persistent record of the session. Use `get_transcript_path()` to retrieve the configured path.

## `AgentNode`

A named controller configuration.

### Constructor

```python
AgentNode(
    name: str,
    system_prompt: str = "",
    append_system_prompt: str = "",
    tools: list[str] = None,
    allowed_tools: list[str] = None,

)
```

### Methods / properties

- `start() -> bool`
- `stop()`
- `send(text: str, timeout: float = 60) -> Optional[str]`
- `alive`

## `MultiAgentRouter`

A thin coordination layer for multiple named nodes.

### Constructor

```python
MultiAgentRouter()
```

### Methods

#### `add(node: AgentNode) -> MultiAgentRouter`
Registers a node.

#### `start_all(parallel: bool = True)`
Starts all nodes, optionally in parallel.

#### `stop_all()`
Stops all nodes and clears the registry.

#### `send(agent_name: str, message: str, timeout: float = 60) -> Optional[str]`
Sends a message to one named node.

#### `route(message: str, to: str, wrap: str = "{message}", timeout: float = 60) -> Optional[str]`
Wraps and forwards a message to another node.

#### `parallel_send(message: str, agent_names: list[str], timeout: float = 90) -> dict[str, str]`
Sends the same message to several nodes in parallel.

#### `get_ctrl(agent_name: str) -> ClaudeController`
Returns the underlying controller for advanced use.

### Context manager

`MultiAgentRouter` implements `__enter__` / `__exit__` and stops all nodes on exit.

## `runtime` module

Runtime introspection for the `claude` CLI binary.

### `find_claude_binary(cli_path: str = "claude") -> str | None`

Resolves a CLI name to an absolute path via `shutil.which`. Returns `None` if not found.

### `get_claude_version(binary_path: str) -> str | None`

Runs `claude --version` and returns the first line on success. Returns `None` on failure.

### `check_claude_available(cli_path: str = "claude") -> ClaudeRuntimeInfo`

Checks whether the CLI binary is available. Returns a `ClaudeRuntimeInfo` dataclass. Raises `ClaudeBinaryNotFound` if the binary is not in PATH.

### `ClaudeRuntimeInfo`

```python
@dataclass
class ClaudeRuntimeInfo:
    cli_path: str       # original CLI name used
    binary_path: str | None  # resolved absolute path
    version: str | None       # from --version, or None
    available: bool
```

## `exceptions` module

All exceptions inherit from `ClaudeError` (a `RuntimeError` subclass).

### Exception hierarchy

- `ClaudeError` — base class; inherits from `RuntimeError`
- `ClaudeBinaryNotFound` — binary not found in PATH
- `ClaudeStartupError` — subprocess failed to start
- `ClaudeTimeoutError` — operation exceeded timeout
- `ClaudeSendConflictError` — concurrent send to same controller

## API design guidance

The public API should remain small.

If a proposed addition introduces concepts like:

- jobs,
- pipelines,
- memory stores,
- governance,
- dashboards,
- or workflow graphs,

it probably belongs outside this library.
